# 校园直聘平台 - 需求规格说明书

## 功能概述

校园直聘平台是一个面向高校、企业和学生的综合性招聘服务平台。平台连接高校人才与企业需求，提供便捷的招聘求职服务。系统支持教师端、企业端、学生端和运营管理端四个角色，涵盖招聘发布、简历管理、在线聊天、宣讲会/双选会管理、视频面试、数据统计等核心功能，旨在提升校园招聘效率，优化求职体验。

## 用户故事

### 教师端用户故事

**US-T-001**: 作为教师，我希望能够查看和分析学生的活跃度数据，以便了解学生的求职状态和参与度。
- **验收标准**:
  - 教师可以按时间和院系条件过滤学生活跃度数据
  - 系统能够展示活跃度统计图表
  - 数据支持导出功能

**US-T-002**: 作为教师，我希望能够发布和管理双选会、宣讲会，以便组织校园招聘活动。
- **验收标准**:
  - 教师可以创建、编辑、发布双选会和宣讲会
  - 可以查询报名企业和学生信息
  - 可以邀请企业参加活动

**US-T-003**: 作为教师，我希望能够为学生推荐合适的职位，以便帮助学生找到理想工作。
- **验收标准**:
  - 教师可以为某个职位推荐管辖内的学生
  - 推荐信息能够及时通知学生
  - 可以查看推荐状态和反馈

### 企业端用户故事

**US-E-001**: 作为企业HR，我希望能够发布和管理招聘职位，以便吸引合适的人才。
- **验收标准**:
  - 企业可以创建、编辑、发布、关闭职位
  - 支持职位草稿保存和审核流程
  - 可以设置职位关键词和标签
  - 可以查看职位数据统计

**US-E-002**: 作为企业HR，我希望能够搜索和查看学生简历，以便找到合适的候选人。
- **验收标准**:
  - 企业可以搜索学生简历
  - 可以查看简历详情
  - 可以收藏、标记、分享简历
  - 可以发送面试邀请和宣讲会邀请

**US-E-003**: 作为企业HR，我希望能够与学生和学校在线聊天，以便及时沟通招聘事宜。
- **验收标准**:
  - 支持文字、图片、位置等消息类型
  - 可以查看学生简历和学校主页
  - 可以在聊天中发起面试邀请
  - 支持简历下载功能

**US-E-004**: 作为企业HR，我希望能够管理宣讲会和双选会，以便参与校园招聘活动。
- **验收标准**:
  - 可以查看和申请双选会
  - 可以创建和管理宣讲会
  - 支持双选会签到功能
  - 可以查看活动数据统计

### 学生端用户故事

**US-S-001**: 作为学生，我希望能够创建和管理个人简历，以便向企业展示我的能力和经历。
- **验收标准**:
  - 学生可以创建、编辑、删除简历
  - 支持多份简历管理
  - 可以设置默认简历
  - 简历支持多种格式导出

**US-S-002**: 作为学生，我希望能够搜索和申请合适的职位，以便找到理想工作。
- **验收标准**:
  - 学生可以搜索企业和职位
  - 可以查看职位详情
  - 可以一键投递简历
  - 可以收藏感兴趣的职位

**US-S-003**: 作为学生，我希望能够参加宣讲会和双选会，以便了解企业信息和招聘需求。
- **验收标准**:
  - 学生可以搜索和查看宣讲会/双选会信息
  - 可以报名参加活动
  - 支持扫码签到功能
  - 可以查看活动日程

**US-S-004**: 作为学生，我希望能够接收和管理面试邀请和offer，以便及时了解求职进展。
- **验收标准**:
  - 学生可以接收面试邀请
  - 可以查看和管理面试安排
  - 可以接收和查看offer信息
  - 支持视频/语音面试功能

### 运营管理端用户故事

**US-O-001**: 作为运营管理员，我希望能够配置和管理权益系统，以便控制平台功能权限。
- **验收标准**:
  - 可以配置基础权益和权益套餐
  - 可以查询用户权益信息
  - 可以管理权益购买和消费记录
  - 支持权益数据分析

**US-O-002**: 作为运营管理员，我希望能够查看和分析平台数据，以便了解平台运营状况。
- **验收标准**:
  - 可以查看用户数据分析
  - 可以查看用户活跃度分析
  - 可以查看用户行为分析
  - 可以查看权益收益分析

## 验收标准

### 功能验收标准

1. **用户认证和权限管理**
   - 所有用户角色支持注册、登录、密码重置功能
   - 企业用户支持企业认证和个人身份认证
   - 教师用户支持院校实名认证
   - 权限系统能够正确控制各角色的功能访问

2. **核心业务流程**
   - 企业可以发布职位，学生可以搜索和申请职位
   - 企业可以查看学生简历，学生可以管理简历
   - 支持在线聊天功能，消息能够及时送达
   - 宣讲会和双选会流程完整，支持报名、签到、管理功能

3. **数据统计和分析**
   - 各端数据报表功能正常
   - 数据统计准确，支持多维度分析
   - 数据支持导出功能

### 性能验收标准

1. **响应时间**
   - API接口响应时间 < 500ms（95%请求）
   - 页面加载时间 < 2s
   - 数据库查询时间 < 100ms

2. **并发性能**
   - 支持1000+并发用户
   - 系统可用性 > 99.9%

3. **资源使用**
   - 前端资源压缩后 < 2MB
   - 图片资源支持懒加载和CDN加速

### 安全验收标准

1. **数据安全**
   - 用户密码加密存储
   - 敏感数据传输使用HTTPS
   - SQL注入防护
   - XSS攻击防护

2. **权限安全**
   - 接口权限验证
   - 数据权限隔离（教师只能查看管辖学生数据）
   - 文件上传类型和大小限制

3. **合规要求**
   - 用户隐私保护
   - 数据备份和恢复机制
   - 操作日志记录

## 性能/安全/合规约束

### 性能约束

- 系统需要支持至少1000并发用户
- 数据库查询需要优化，关键查询需要索引
- 前端资源需要压缩和CDN加速
- 图片和文件需要支持异步上传

### 安全约束

- 所有API接口需要身份认证
- 敏感操作需要二次验证
- 用户数据需要加密存储
- 需要定期进行安全审计

### 合规约束

- 遵守《个人信息保护法》
- 遵守《网络安全法》
- 用户数据需要定期备份
- 需要提供数据导出功能

## 不在范围内

以下功能不在本次开发范围内：

1. **移动端原生APP开发**（仅支持Web端和响应式设计）
2. **第三方支付集成**（权益购买功能暂不实现支付）
3. **实时视频会议系统**（视频面试功能使用第三方SDK）
4. **AI智能推荐算法**（推荐功能使用基础规则匹配）
5. **多语言支持**（仅支持中文）
6. **消息推送服务**（仅支持站内消息）
