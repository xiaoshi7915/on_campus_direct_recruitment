# 快速测试指南

## 🚀 服务启动状态

**后端服务：** http://localhost:5001
- API文档：http://localhost:5001/docs
- 健康检查：http://localhost:5001/health

**前端服务：** http://localhost:5173
- 访问地址：http://localhost:5173

## 📋 快速测试流程

### 第一步：验证服务运行
1. 打开浏览器访问 http://localhost:5173
2. 应该能看到登录页面

### 第二步：测试教师审批功能
1. **管理员登录**
   - 用户名：`admin`
   - 密码：`admin123`
   - 登录后进入"教师审批"页面

2. **审批教师**
   - 在待审批列表中应该能看到 `pending_teacher`
   - 点击"通过"按钮
   - 验证审批成功

### 第三步：测试审批流程
1. **主账号教师登录**
   - 用户名：`main_teacher`
   - 密码：`teacher123`
   - 登录后进入"双选会管理"或"宣讲会管理"

2. **创建待审批的双选会/宣讲会**
   - 点击"创建"按钮
   - 填写信息并提交
   - 验证状态为"待审批"

3. **审批双选会/宣讲会**
   - 在列表中点击"审批"按钮
   - 选择"通过"并提交
   - 验证状态更新为"已发布"

### 第四步：测试子账号管理
1. **进入子账号管理页面**
   - 导航栏点击"子账号管理"
   - 查看子账号列表

2. **创建子账号**
   - 点击"创建子账号"
   - 填写信息：
     - 用户名：`sub_teacher_001`
     - 密码：`sub123456`
     - 真实姓名：`子账号教师`
   - 提交创建
   - 验证子账号出现在列表中

### 第五步：测试学校信息
1. **进入我的院校页面**
   - 导航栏点击"我的院校"
   - 验证学校信息正常显示
   - 验证没有500错误

## 🐛 问题排查

如果遇到问题，请检查：

1. **服务未启动**
   ```bash
   # 检查后端
   curl http://localhost:5001/health
   
   # 检查前端
   curl http://localhost:5173
   ```

2. **API错误**
   - 打开浏览器开发者工具（F12）
   - 查看Network标签页
   - 检查失败的API请求
   - 查看Console标签页的错误信息

3. **数据不存在**
   ```bash
   # 重新创建测试数据
   cd backend
   python scripts/create_test_data.py
   ```

4. **权限问题**
   - 确认使用正确的账号登录
   - 检查用户类型是否正确
   - 检查token是否有效
